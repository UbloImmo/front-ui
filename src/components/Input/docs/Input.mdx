## Controls

All inputs can be managed on their state, appearance, and interactions with the users.

### Disable

Inputs are active by default. Deactivate an input, and prevent the user from entering any data, can be done with the property `disabled`.

When set to true, the input become read-only, and is grayed out to inform users that they can't enter value there.

<Canvas of={InputStories.Disabled} />

### Error

Controlling the validity of an input can be made with the `error` property.

When set to true, the input's style changes (red border color) and alert visually the user that the entered data is invalid.

<Canvas of={InputStories.Error} />

### Placeholder

While the placeholder property is optional, including it in the input field is useful for guiding users on the expected value. This placeholder remains visible until users enter a value.
