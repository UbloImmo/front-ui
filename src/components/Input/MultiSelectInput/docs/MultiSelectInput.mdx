## Usage

`MultiSelectInput` is a variant of [SelectInput](?path=/docs/components-forms-input-selectinput-usage--docs) that allows the user to select multiple options from a list.
It displays its selected options using a regular [Chip](?path=/docs/components-actions-chip-usage--docs) component.

### Un/Selecting options

To select an option, simply click on it in the dropdown list. The selected option will appear as a chip above the input field. multiple options can be selected, and each will be displayed as a separate chip.

Unlike [SelectInput](?path=/docs/components-forms-input-selectinput-usage--docs), this input cannot be made searchable.

To unselect an option, you can either click on the <Icon name="XLg" size="s-3" /> icon on the chip to remove it.

> Clicking on a selected option in the dropdown list will **not** unselect it.

### Disabled options

You can disable specific options by adding a `disabled` property to them.
Disabled options cannot be selected not unselected and are visually distinct from selectable options.

> All options will be disabled if the `disabled` property is set on the `MultiSelectInput` component.

<Canvas of={MultiSelectInputStories.DisabledOptions} />

### Loading state

This input shows the loading state of the options by displaying a loading animation. Specifically, a **BouncingBalls** [Loading](?path=/docs/components-feedbacks-loading-usage--docs) component is displayed within the input while the options are being loaded.

<Canvas of={MultiSelectInputStories.LoadingOptions} />

### Customization

#### Custom options

This input supports rendering arbitrary components for its options only.

> Extra data can be passed each option, and as such to custom components through the `option.extraData` property.

The input's height will scale according to its option's size. It is currently not possible to customize the appearance of the inner search input
that is displayed on focus when the `searchable` property is enabled.

> The active state can be consumed inside any custom component through the `props.active` property.

<Canvas of={MultiSelectInputStories.CustomComponents} />

#### Custom control icon

`MultiSelectInput` uses the `CaretDownFill` [Icon](?path=/docs/components-iconography-icon-usage--docs) (<Icon name="CaretDownFill" />) by default as its _control icon_.
To customize it, provide the name of your icon in the optional `controlIcon` property.

Please refer to the [Icon Repository](?path=/docs/components-iconography-icon-repository--docs) for a complete list.
