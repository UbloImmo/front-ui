## Typing

In order to make the List component type safe while still being flexible, we needed it to be _generic_.

That means you need to define a type for the items you want to display. 

> _We'll be using this `Pokemon` type in the examples below._

```typescript
type Pokemon = {
  id: number;
  name: string;
  base_experience: number;
  is_default: boolean;
  order: number;
  weight: number;
  types: {
    slot: number;
    type: {
      id: number;
      name: string;
    };
  }[];
};
```


## Data Provider

The component expects a `dataProvider` hook to be provided.
**It is responsible for fetching / refetching, filtering and sorting the data.**
Your custom hook will the be ran internally, and therefore drive what / how data is displayed.

<Callout color="primary">
  If you need to implement custom logic data / filtering / sorting for a specific list, this is the place to do it.
</Callout>

Don't worry, we provide a basic `useStaticDataProvider` hook, which is a good starting point that aims to cover most use cases.

```tsx
import { useStaticDataProvider, type UseDataProviderFn } from "@ubloimmo/uikit";

// fetches a list of pokemon
const fetchPokemon = async (): Promise<Pokemon[]> => {
  // ...
  return [];
}

// create a data provider hook that provides a fetcher
// and forwards the setData callback to the static data provider
const usePokemonData: UseDataProviderFn<Pokemon> = (setData) => {
  return useStaticDataProvider(fetchPokemon, setData);
} 
```

## Behavior

Now that we know what our data looks like and have a way to manage it, we can start configuring how the user can interact with it.

Import the `useListConfig` hook to configure a list&apos;s options and behavior.
It yieds a `config` object and various typed methods that mutate it.

> This `config` object is the one and only property supplied to either the `List` component or its related `ListProvider`.

### Barebones Configuration

If all you need to do is display data in a list, this is the simplest way to do it.

```tsx
import { useListConfig } from "@ubloimmo/uikit";

/**
 * Custom hook that configures the list
 */
const usePokemonListConfig = () => {
  // we pass our data provider hook to the useListConfig hook, without calling it
  const {
    config
  } = useListConfig<Pokemon>(usePokemonData);
  
  // and return the config object
  return config;
}
```

## Configuration methods

When there is a lot of data, it is often useful to allow the user to filter it.

These filters and options are, _you guessed it_, built using the `useListConfig` hook and its methods.

<Callout color="warning" icon="ExclamationTriangleFill">
  We recommend wrapping configuration method calls in a `useStatic` hook, to ensure they are only called once.
  **Expect infinite render loops otherwise.**
</Callout>

### Option matches

- [match()](?path=/docs/components-list-configuration-methods-match--docs): Builds and returns a _filter option match_ for the list's data.
- [matches()](?path=/docs/components-list-configuration-methods-match--docs): Builds and returns a **list** of _filter option matches_ for the list's data.

### Options

- `option()`
- `options()`
- `async.options()`

### Filters

- `filter()`
- `async.filter()`

### Filter presets

- `filterPreset()`
- `async.filterPreset()`
