## Writing a new animation

This component is extensible, and can easily render more animations. Its types are dynamic and flexible

### Structure

Additional animations can be created by creating a new directory under `src/components/Loading/animations`.

It must comply with the following structure.

```txt
[Anim]
├── [Anim].animation.tsx
├── [Anim].styles.ts
├── [Anim].module.scss
└── index.ts
```

> Please replace `[Anim]` with any name of your choosing.

### Contents

#### `[Anim].animation.ts`

Defines a single React component, named `[Anim]`, that takes `LoadingAnimationProps` and renders your animation.

The animation itself can be any HTML Element, though SVGs are recommended.
Make sure to include the `data-testid={props.testId}` attribute in your markup.

> This file should **only** export your animation component.
>
> This is crucial for the dynamic types to work.

##### Example

```tsx
// [Anim]/[Anim].animation.ts

// animation styles
import { useAnimStyles } from "./[Anim].styles";
// type imports
import type { LoadingAnimationProps } from "../../Loading.types";

/**
 * Renders an [Anim] loading animation
 *
 * @param {LoadingAnimationProps} props - the loading animation props.
 * @return {JSX.Element} the rendered spinner component
 */
export const Anim = (props: LoadingAnimationProps): JSX.Element => {
  // compute styles
  const { style, className } = useStyleProps(props);
  // Apply styles to your animation
  // The returned element & structure is up to you
  return <svg style={style} className={className} data-testid={props.testId} />;
};
```

#### `[Anim].styles.ts`

Links your animation's properties to its SCSS module.

Usually exports a function of returning both `className` and `style` properties.

##### Example

```tsx
// [Anim]/[Anim].styles.ts

// type imports
import type { LoadingAnimationProps } from "../../Loading.types";
// stylesheet import
import styles from "./[Anim].module.scss";
// utility imports
import { useCssClasses, useCssVariables } from "@utils";

export function useAnimStyles({size, color}: LoadingAnimationProps) {
  const className = useCssClasses(styles);
  const style = useCssVariables({size, color});
  return { className, style };
}
```

#### `[Anim].module.scss`

SCSS module that contains both styles and keyframes needed for you animation.

Dynamic style properties such as `size` and `color` may be passed to the animation via css variables.

##### Example

```scss
// [Anim]/[Anim].module.scss

@keyframes animate {
  from {
    transform: scale(0);
    opacity: 1;
  }
  to {
    transform: scale(1);
    opacity: 0;
  }
}

.animation {
  // define default size & color
  --size: var(--s-4);
  --color: var(--primary-base);

  // apply variables & style your animation however you want
  width: var(--size);
  height: var(--size);
  background: var(--color);
  animation: animation 1800ms var(--bezier) infinite running
}
```

#### `index.ts`

Your `index.ts` file should then only need to contain the following

```tsx
// [Anim]/index.ts

export * from "./Anim.animation";
```

## Registering a new animation.

### Linking

Your animation is now ready to be linked to `Loading` component to be rendered.

To do so, edit the `src/components/Loading/animations/index.ts` file
and add a new export pointing to your freshly made animation's directory.

```tsx
// animations/index.ts

// existing exports
export * from "./BouncingBalls";
export * from "./Spinner";
// a new export
export * from "./[Anim]";
```

**That's it!**

### Trying it out

You've made it all the way here ? **Congrats !**

Your new animation's name should now be valid when passed as `Loading`'s `animation` prop.

To try your new animation, head over to this component's [Default story](?path=/story/components-feedbacks-loading-stories--default) and type your new animation's in the controls.
