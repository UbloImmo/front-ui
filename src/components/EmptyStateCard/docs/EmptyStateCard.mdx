## Usage

EmptyStateCard is used to display an empty state in most use cases.
It renders a single title in a card by default, but may also display an asset, a one of two descriptions, allowing for custom JSX content.

### Text content

#### Title

By default, EmptyStateCard renders a title using the `empty` status translation.

This can be overridden by providing a string or a `ReactNode` as `title` property.

> Any title gets rendered in a Heading component.

<Canvas of={EmptyStateCardStories.Titles} />

#### Descriptions

Both `description` and `editingDescription` properties are optional and support any `ReactNode` value.

`editingDescription` will only replace `description` if: 
 - it is provided
 - the component is rendered withing a Form
 - that form is in edit mode

> Only one description may be rendered at a time.

<Canvas of={EmptyStateCardStories.Descriptions} />

### Customization

#### Assets

The `asset` property is used to reference an asset to be renderered above the text content. Defaults to `null`.

Currently available asset names are: {Object.keys(EmptyStateCardAssets).join(", ")}.

> More assets may be added in the future.

<Canvas of={EmptyStateCardStories.Assets} />

#### Colors

The default `primary` color is used for the title and asset.
It can be overridden by providing a valid `ColorKey` as `color` property.

<Canvas of={EmptyStateCardStories.Colors} />

#### Icons

Some asset may render an icon. What icon gets rendered may be defined by providing a valid `IconName` as `icon` property.

<Canvas of={EmptyStateCardStories.Icons} />

#### Transparent

Since EmptyStateCard may be embedded in any container, it may be useful to only render its contents (text & asset), but not its container.

Setting the `transparent` property to `true` will remove the card's background and shadow, while keeping the padding.

<Canvas of={EmptyStateCardStories.Transparent} />
