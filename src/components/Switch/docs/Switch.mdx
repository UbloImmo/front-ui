## Usage

The `Switch` component is commonly used to enable users to enable users to toggle features and / or options.

### Activate the switch

Based on selecting or deselecting options, features, etc., like `ComboBox`, `Switch` however will be used to toggle on and off the selectable item, while `ComboBox` allows users to select a single or many items in a list of selectOptions.

> Check [ComboBox](?path=/docs/components-forms-combobox-usage--docs)

> `Switch` state is handled with the `active` property.

<Canvas of={SwitchStories.Active} />

### Disabled

The `Switch` component includes an optional `disabled` property, which can be set to true when you want to disable the toggling functionality.

> Give it a try, those Switches below won't toggle if you click on them

<Canvas of={SwitchStories.Disabled} />

### Adding a helper text

To provide additional information about the active or inactive state of the `Switch` component, you can use the optional `withHelper`, along with the `activeHelperText` and `inactiveHelperText` properties. Here is an example :

```ts
{
  withHelper: true,
  activeHelperText: "Your helper text when the Switch is active",
  inactiveHelperText: "Your helper text when the Switch is inactive",
}
```

When these properties are set, the helper text will be displayed to the right of the `Switch` component.

<Canvas of={SwitchStories.WithHelper} />

#### Helper text positioning

The `helperPosition` property allows you to change the position of the helper text. It defaults to `start`.

```ts
{
  withHelper: true,
  helperPosition: "end",
}
```

> This is useful in avoiding layout shift when the active and inactive helper texts have different lengths.
