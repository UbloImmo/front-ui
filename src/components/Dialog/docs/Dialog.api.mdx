## API Reference

Our Dialog component expects to be used within a parent `DialogProvider`
and interacted with through methods exposed by either `useDialog` or `useDialogManager` hooks.

### Basic example

```tsx
import { DialogProvider, Dialog, useDialog } from "@ubloimmo/uikit";

const App = () => {
  // Wrap your app's content with the Provider
  return (
    <>
      <DialogProvider portalRoot="#dialog-root">
        <Page />
      </DialogProvider>
      {/* This is where any open dialog inside your app will render */}
      <div id="dialog-root" />
    </>
  );
};

/**
 * Any page / component where you want to instanciate a dialog
 */
const Page = () => {
  // The dialog reference is used to control any rendered dialog
  // from anywhere else inside your app
  const dialogReference = "my-dialog";
  // Pass the reference to the useDialog hook to access
  // specific methods for controlling a single dialog
  const { open } = useDialog(dialogReference);
  return (
    <>
      <button title="open dialog" onClick={open}>
        Open
      </button>
      <Dialog reference={dialogReference}>
        <div>Dialog content</div>
      </Dialog>
    </>
  );
};
```

### Setup

#### Portals

All dialogs render their children through a portal to a `portal root` (e.g. an HTML element that can be seperate from your app).

This root can be declared anywhere inside your app, though it is recommended to place it inside your app's `body` in `public/index.html`

> Make sure this root can be easily found by `document.querySelector` (e.g. by giving an `id`).

```html
<!DOCTYPE html>
<html>
  <head>
    <title>My app</title>
  </head>
  <body>
    <div id="app" />
    <div id="dialog-root" />
  </body>
</html>
```

#### Context & Provider

Our Dialog component expects to be used within a parent `DialogProvider`.
It should to be rendered near the root of your DOM tree, preferably inside App.tsx.

<br />
**Import it and wrap your content in it.**

> Alternatively, use `DialogProvider` to wrap a single component if you feel like it.

```tsx
import { DialogProvider } from "@ubloimmo/uikit";

const App = () => {
  return (
    <DialogProvider portalRoot="#dialog-root">
      {/* Your app's content */}
    </DialogProvider>
  );
};
```

> All following examples assume the `Dialog` components are being rendered inside a `DialogProvider`.

### Controlling a single dialog

#### Through its direct parent

If all you need is a simple dialog that reacts to a `boolean` state,
you can pass it to `Dialog`'s `open` property.

`Dialog` will open and close accordingly.

> Each state change triggers the `onClosed` & `onOpened` callbacks if defined.

```tsx
import { useState } from "react";
import { Dialog } from "@ubloimmo/uikit";

const Parent = () => {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <Dialog open={isOpen} reference="my-dialog">
      my dialog
    </Dialog>
  );
};
```

#### Through the `useDialog` hook

The `useDialog` hook exposes a single Dialog's state and control methods.
Use it to control a dialog from sibling or children components.

> Exposed methods include controls (`open()`, `close()`, `toggle()`) as well as lifecycle functions (`register()`, `unregister()`).

```tsx
import { Dialog, useDialog } from "@ubloimmo/uikit";

const Page = () => {
  return (
    <main>
      <Dialog reference="my-dialog">
        <Modal />
      </Dialog>
      <SideControls />
    </main>
  );
};

const Modal = () => {
  const { close } = useDialog("my-dialog");

  return (
    <div>
      <button onClick={close}>Close me</button>
    </div>
  );
};

const SideControls = () => {
  const { open, close, toggle } = useDialog("my-dialog");

  return (
    <aside>
      <button onClick={open}>Open dialog</button>
      <button onClick={close}>Close dialog</button>
      <button onClick={toggle}>Toggle dialog</button>
    </aside>
  );
};
```

### Controlling multiple dialogs

Use the `useDialogManager` hook to control all dialogs within a `DialogProvider`.

> This hook is essentially what runs when using the `useDialog` hook.

```tsx
import { useEffect } from "react";
import { useDialogManager, DialogProvider, Dialog } from "@ubloimmo/uikit";

const AnnouncementDialog = () => (
  <Dialog reference="announcement">{/* Content */}</Dialog>
);

const LogoutDialog = () => <Dialog reference="logout">{/* Content */}</Dialog>;

const Dialogs = () => (
  <>
    <AnnouncementDialog />
    <LogoutDialog />
  </>
);

const App = () => (
  <DialogProvider portalRoot="#modal-root">
    <Dialogs />
  </DialogProvider>
);

const Page = () => {
  const { openDialog } = useDialogManager();

  useEffect(() => {
    openDialog("announcement");
  }, []);

  return (
    <main>
      <h2>My account</h2>
      {/* Content */}
      <button onClick={() => openDialog("logout")}>Log out</button>
    </main>
  );
};
```

### Listening for changes

Provide a callback function to the `onOpened` and / or `onClosed` properties to listen to a dialog's state changes.

> These callbacks are called **_after_** the state changes.

```tsx
import { useState, useCallback } from "react";
import { Dialog, useDialog } from "@ubloimmo/uikit";

const Page = () => {
  const { open } = useDialog("my-dialog");

  const onDialogVisibilityChanged = useCallback((opened: boolean) => {
    console.log(`Dialog is ${opened ? "open" : "closed"}`);
  }, []);

  return (
    <main>
      <button onClick={open}>Open dialog</button>
      <Dialog
        reference="my-dialog"
        onOpened={() => onDialogVisibilityChanged(true)}
        onClosed={() => onDialogVisibilityChanged(false)}
      >
        <Modal />
      </Dialog>
    </main>
  );
};

const Modal = () => {
  return <div>Modal content</div>;
};
```
