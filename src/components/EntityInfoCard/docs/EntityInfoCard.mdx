## Usage

Need to display an entity's relevant information, state and / or trigger simple actions on it ?
_Look no further!_

### Content customization

#### Examples

Here are some of the ways one could use `EntityInfoCard` to provide a clutter-free view into an entity's data.

<Canvas of={EntityInfoCardStories.Examples} />

#### Information display

EntityInfoCard can display information in various forms, through the use of several global components:

- [InfoBox](?path=/docs/components-entity-infobox-usage--docs)
- [CopyClipboardInfoCard](?path=/docs/components-entity-copyclipboardinfocard-usage--docs)
- [ContextInfoCard](?path=/docs/components-entity-contextinfocard-usage--docs)
- [ContextLine](?path=/docs/components-entity-contextline-usage--docs)
- [Badge](?path=/docs/components-states-badge-usage--docs)

> Combine some or all of them to create a disctinct layout for each entity.
>
> Each of the following examples use the same underlying data.

<Canvas of={EntityInfoCardStories.InfoStyles} />

#### Custom Content

When all is lost and you need to display some info using custom means, pass a regular element / component into `EntityInfoCard`'s `children`.

<Canvas of={EntityInfoCardStories.MainContent} />

#### Sections


By default, `EntityInfoCard` will display its content in a single section.
If need be, you can split its content into multiple sections by providing an array of `EntityInfoCardSectionProps` to its `sections` property.

> These sections may contain one or more data type supported by `EntityInfoCard`.

<Canvas of={EntityInfoCardStories.Sections} />


### Header customization

#### State

`EntityInfoCard` makes use of a [StateIndicator](?path=/docs/components-states-stateindicator-usage--docs) in its header.
As such, its `state` property accepts all of the underlying component's properties.

<Canvas of={EntityInfoCardStories.State} />

#### Account Balance

`EntityInfoCard` can also display an [AccountBalance](?path=/docs/components-entity-accountbalance-usage--docs) in its header.

<Canvas of={EntityInfoCardStories.AccountBalance} />

#### Action Icon

A standard [ActionIcon](?path=/docs/components-actions-actionicon-usage--docs) is rendered when the `actionIcon` property is provided.

<Canvas of={EntityInfoCardStories.ActionIcon} />

#### Context Menu

One sometimes needs to display additional secondary actions in a context menu.
`EntityInfoCard` will render a [ContextMenu](?path=/docs/components-actions-contextmenu-usage--docs) in its header if its `contextMenu` property is provided.

<Canvas of={EntityInfoCardStories.ContextMenu} />
