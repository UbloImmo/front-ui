## Usage

Need to create a collapsible side navigation menu for entity-specific actions and navigation?
The `SideEntityMenu` provides a space-efficient, hover-expandable sidebar perfect for entity contexts.

### Basic Usage

The component displays as a narrow icon-only menu by default and expands to show full text labels when hovered.

<Canvas of={SideEntityMenuStories.Default} />

### Navigation Links

#### Title Configuration

The menu can display a title header with a configurable icon. Use the `title` prop to set the text and `titleIcon` to customize the icon.
This header is not clickable and serves as information for the current entity.

<Canvas of={SideEntityMenuStories.WithNonClickableTitle} />

#### Menu Links

The primary navigation items are defined through the `menuLinks` array. Each link supports:

- Navigation URLs (`to` property)
- Click handlers (`onClick` property)
- Icons, titles, and various states
- Pinned items for priority actions

<Canvas of={SideEntityMenuStories.WithoutBackLinks} />

#### Back Links

Back navigation links appear in the header section and provide context for the current entity view.

<Canvas of={SideEntityMenuStories.Default} />

### Interactive Features

#### Pinned Items

Important actions can be pinned to the top of the menu with optional helper text.
The pinned section is separated by spacing for visual hierarchy.

> Pinned items appear first in the menu.

#### Error States

Menu items can display error states with visual indicators to highlight problematic actions.

<Canvas of={SideEntityMenuStories.WithErrorState} />

#### Disabled State

The entire menu or individual items can be disabled to prevent interaction.

<Canvas of={SideEntityMenuStories.WithIndividualDisabledItems} />

#### Hidden Items

Menu items can be conditionally hidden based on user permissions or context.

<Canvas of={SideEntityMenuStories.WithHiddenItems} />

#### Active State Control

The menu supports both automatic URL-based active detection and controlled active state via the `activeItem` prop.
When using the controlled approach, the active indicator smoothly animates between menu items.

### Navigation Options

The `SideEntityMenu` supports different navigation patterns depending on your application's routing approach:

#### Standard HTML Navigation (Default)

By default, menu items use standard HTML anchor tags with `href` attributes for navigation.
This is suitable for server-side rendered applications or when you want browser-native navigation.

#### React Router Integration

For React Router applications, use the `navigate` prop to provide programmatic navigation:

<Canvas of={SideEntityMenuStories.WithReactRouterNavigation} />

```typescript
import { useNavigate } from 'react-router-dom';

const MyComponent = () => {
  const navigate = useNavigate();

  return (
    <SideEntityMenu
      menuLinks={menuLinks}
      navigate={navigate}
      activeItem={currentPath}
    />
  );
};
```

#### Custom Click Handlers

Individual menu items can override navigation with custom `onClick` handlers:

```typescript
const menuLinks = [
  {
    title: "Custom Action",
    icon: "Settings",
    to: "/settings",
    onClick: () => {
      // Custom logic here
      console.log("Custom action triggered");
    },
  },
];
```

**Navigation Priority**: `navigate` prop > `onClick` handler > default HTML navigation

### Menu Link Properties

Each menu link supports the following configuration:

- **`title`**: Display text (shown on hover)
- **`icon`**: Icon name from the icon library
- **`to`**: Navigation URL (for links)
- **`onClick`**: Click handler (for actions)
- **`pinned`**: Whether item appears in priority section
- **`error`**: Error state styling (f.e. when necessary info missing after unit creation)
- **`disabled`**: Individual item disable state (f.e. can be the case when creating a invoice model)
- **`hidden`**: Hide item conditionally (f.e. can be the case when feature flagged)
