## Usage

`ActionIcon` is the smallest counterpart to our [Action](?path=/docs/components-action-usage--docs) component.

> Use it when offering "classic", *easily recognizable* actions to our users (e.g. delete, copy, archive).

### Accessibility

#### Icons

As with most other customizable components, ActionIcon's `icon` property directly maps to is underlying `Icon` component's `name` property.

> Please refer to the [Icon Repository](?path=/docs/components-icon-repository--docs) for a complete list.

<Canvas of={ActionIconStories.Icons} />


#### Titles

This button does not display a text label. Therefore, its `title` property is **required**.

> Hover over the following examples to see their accessible titles.

<Canvas of={ActionIconStories.Titles} />



### Sizing

Sometimes actions are so secondary that they only need to fill the smallest possible amount of space.

Set the `size` property to `s` to display an even smaller variant of ActionIcon. Defaults to `m`.

<Canvas of={ActionIconStories.Sizes} />


### Destructive actions

Readability and recognition are essential when presenting our users with an unlabed action.
<br/>
Inform them about a potentially *risky* action by setting its `color` property to `error`. It defaults to `primary` otherwise.

*This might be simplified to a boolean property if no other colors are to be added to the component*

> Hover over each variant to experience the differences.

<Canvas of={ActionIconStories.Colors} />

### Disabled

ActionIcons are inherently buttons. We need a way to disable them.

Set its `disabled` property to `true` to do just that.
